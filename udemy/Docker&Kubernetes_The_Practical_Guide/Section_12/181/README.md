# 181. Kubernetes: 요구 설정 & 설치 단계

k8s를 시작하려면 우선 클러스터가 필요하다.

실제로 마스터 노드가 있는 클러스터는 실제로 여러 머신에 분산되어 항상 구동되어 실행될 수 있다.

마스터 노드가 설정된 클러스터에서 실행되는 하나의 가상 인스턴스가 필요하고 하나 이상의 워커 노드가 필요하다. 

따라서 마스터 노드에는 API 서버와 스케줄러 등이 설치되어야 하고, 워커 노드에는 Docker 가 설치되어야 하고 kubelet이 설치되어야 한다.

그리고 kubectl 이라는 kube control 도구도 필요하다. kubectl은 실제로 새 deployment 생성/삭제/변경과 같은 명령을 클러스터에 보내는데 사용하는 도구이다. kubectl 을 사용하면 개발자(또는 관리자)가 마스터 노드에 명령을 보낼 수 있으며, 마스터 노드는 워커 노드와 관련하여 필요한 모든 작업을 수행한다.

예를 들어, 주어진 컨테이너의 더 많은 pod을 생성하라는 명령을 보낸 경우 (주어진 컨테이너의 더 많은 인스턴스를 생성하라는 경우), 마스터 노드와 워커 노드가 상호 작용하여 이러한 추가 pod이 생성된다.

cluster는 기술적인 인프라이다. 

kubectl은 해당 인프라와 통신하고 k8s 설정과 통신하기 위한 통신 장치이다.
(kubectl != API Server && kubectl != master node)

마스터 노드는 기본적으로 명령을 적용하고 명령이 올바르게 실행되도록 하는 클러스터에 존재한다. kubectl은 명령을 내리는 도구다.
